# Security Policy

## Supported versions

We apply security fixes to the default branch (`main`) as feasible. There are no numbered LTS releases yet; deploy from `main` or a recent tag if you add them later.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead, report privately through one of these channels:

1. **GitHub Security Advisories** (preferred if enabled on the repository):  
   Repository → **Security** → **Advisories** → **Report a vulnerability**.

2. **If Advisories are unavailable**: open a draft-only discussion or email the repository owner via their GitHub profile contact options, with subject line containing `[SECURITY] ai-frontier-tracker`.

Include:

- Description of the issue and potential impact
- Steps to reproduce (if known)
- Affected component (e.g. fetcher, API route, dependency version)

We aim to acknowledge reports within a few business days. Thank you for helping keep users safe.
